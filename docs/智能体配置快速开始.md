## 🚀 5分钟快速上手

### 第一步：启动系统

```bash
# 方式1：一键启动（推荐）
START_ALL.bat

# 方式2：分别启动
# 终端1 - 启动后端
python backend/server.py

# 终端2 - 启动前端
cd alpha-council-vue
npm run dev
```

### 第二步：打开配置面板

1. 访问 `http://localhost:8080`
2. 点击顶部导航栏的 **🤖 智能体** 按钮
3. 配置面板将弹出

### 第三步：选择配置方案

点击以下任一方案：

| 方案 | 适用场景 | 时间 | 质量 |
|------|---------|------|------|
| ⚡ 最小化 | 快速决策 | 45秒 | 80% |
| ⚖️ 平衡 | 日常分析 | 75秒 | 93% |
| 🎯 完整 | 深度研究 | 120秒 | 100% |

### 第四步：开始分析

1. 关闭配置面板
2. 输入股票代码（如：600519）
3. 点击 **🚀 开始分析**
4. 系统将根据你的配置运行智能体

- 🏭 行业轮动分析师
- 💰 资金流向分析师

- 💬 社交媒体分析师 - 舆情监控
- ⚡ 市场动能总监 - 技术面整合
- 🛡️ 系统性风险总监 - 系统风险

### 2. 配置影响预览

配置面板底部实时显示：

```
┌─────────────────────────────────────┐
│ 启用智能体: 15/21                    │
│ 预计时间: 75秒                       │
│ 分析质量: 93%                        │
```

💡 **提供降级方案**

## 🐛 常见问题

### Q1: 为什么核心智能体不能禁用？

**A**: 核心智能体是系统运行的基础，禁用会导致分析流程中断。例如：
- 没有"新闻分析师"，多空辩论无法进行
- 没有"研究部经理"，无法生成投资建议
- 没有"风控部经理"，无法评估风险

### Q2: 禁用智能体会影响哪些功能？

**A**: 系统会自动检查并提示：

```
禁用 "社交媒体分析师"
→ 影响：行业轮动分析将缺少社交媒体维度
→ 降级：使用新闻分析代替

禁用 "中国市场专家"
→ 影响：宏观分析将使用通用市场分析
→ 降级：对A股的专项分析减弱
```

### Q3: 配置保存在哪里？

**A**: 配置保存在 `backend/data/agent_config.json`

```json
{
  "news_analyst": true,
  "fundamental": true,
  "technical": true,
  ...
}
```

可以手动编辑此文件，但建议通过配置面板操作。

### Q4: 如何重置配置？

**A**: 三种方式：

1. **配置面板** → 点击"重置"按钮
2. **应用方案** → 选择"平衡配置"
3. **删除文件** → 删除 `backend/data/agent_config.json`

### Q5: 配置会立即生效吗？

**A**: 
- ✅ 保存后立即生效
- ✅ 下次分析使用新配置
- ❌ 不影响正在进行的分析

## 🎓 最佳实践

### ✅ 推荐做法

1. **日常使用平衡配置**
   - 兼顾速度和质量
   - 满足大多数场景

2. **根据场景调整**
   - 快速看盘用最小化
   - 重要决策用完整

3. **关注影响预览**
   - 确保质量评分 >85%
   - 控制时间在可接受范围

4. **定期检查配置**
   - 确保配置符合当前需求
   - 及时调整优化

### ❌ 避免做法

1. **盲目禁用智能体**
   - 不了解影响就禁用
   - 可能导致分析质量下降

2. **过度追求速度**
   - 禁用过多智能体
   - 牺牲分析质量

3. **忽略警告提示**
   - 不看依赖关系警告
   - 可能导致功能异常
